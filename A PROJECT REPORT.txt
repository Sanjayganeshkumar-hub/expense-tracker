## CHAPTER 1: INTRODUCTION

### 1.1 Overview

The Personal Expense Tracker is a comprehensive web-based application designed to help individuals manage their personal finances effectively. In today's digital age, where financial transactions occur frequently and through multiple channels, keeping track of income and expenses has become increasingly challenging. This application provides users with a centralized platform to record, monitor, and analyze their financial activities.

The application leverages modern web technologies to deliver a seamless user experience while maintaining robust security and data integrity. Built on the MERN stack (MongoDB, Express.js, Node.js), it combines the flexibility of NoSQL databases with the power of JavaScript-based server and client-side programming.

### 1.2 Problem Statement

Many individuals face the following challenges in managing their personal finances:

- **Lack of Awareness:** People often don't know where their money is going due to the absence of systematic tracking
- **Manual Tracking Difficulties:** Traditional methods like paper notebooks or spreadsheets are time-consuming and error-prone
- **No Visual Insights:** Without proper data visualization, it's difficult to understand spending patterns and identify areas for improvement
- **Accessibility Issues:** Existing solutions may require installation, lack cloud synchronization, or have complex interfaces
- **Security Concerns:** Storing financial data requires robust security measures that many simple tools don't provide
- **Category Management:** Difficulty in organizing expenses into meaningful categories for better analysis

### 1.3 Objectives

The main objectives of this project are:

1. To develop a user-friendly web application for tracking personal income and expenses
2. To implement secure user authentication and authorization mechanisms
3. To provide real-time financial overview through an interactive dashboard
4. To enable category-based expense tracking for better financial analysis
5. To visualize spending patterns using charts and graphs
6. To ensure data security through encryption and secure storage
7. To create a responsive design that works across all devices
8. To implement CRUD operations for complete transaction management

### 1.4 Scope of the Project

The scope of this project includes:

**Included:**
- User registration and authentication system
- Transaction management (add, view, edit, delete)
- Dashboard with financial summary
- Category-based expense organization
- Visual data representation using charts

**Future Enhancements (Not in Current Scope):**

- Feature-rich but platform-dependent
- Require installation and updates
- Often expensive with subscription models

**3. Mobile Apps**

- Smartphone-focused interface

**4. Simple Web Tools**

## CHAPTER 4: SYSTEM DESIGN

### 4.1 System Architecture

The application follows a three-tier architecture:

**Presentation Tier (Frontend):**
- HTML5 for structure
- CSS3 for styling and layout
- JavaScript for interactivity
- Handles user interface and user experience
- Communicates with backend via REST APIs

**Application Tier (Backend):**
- Node.js runtime environment
- Express.js web framework
- RESTful API endpoints
- Business logic implementation
- Authentication and authorization
- Data validation and processing

**Data Tier (Database):**
- MongoDB NoSQL database
- Collections for Users and Transactions
- Mongoose for data modeling
- Handles data persistence and retrieval

**Communication Flow:**
1. User interacts with frontend (HTML/CSS/JS)
2. Frontend sends HTTP requests to backend APIs
3. Backend processes requests and interacts with database
4. Database returns requested data
5. Backend sends response to frontend
6. Frontend displays results to user

### 4.2 Database Design

**Collections:**

**1. Users Collection:**
```
{
  _id: ObjectId,
  name: String,
  email: String (unique),
  password: String (hashed),
  createdAt: Date
}
```

**2. Transactions Collection:**
```
{
  _id: ObjectId,
  userId: ObjectId (reference to Users),
  type: String (enum: 'income', 'expense'),
  amount: Number,
  category: String,
  description: String,
  date: Date,
  createdAt: Date
}
```

**Relationships:**
- One-to-Many relationship between Users and Transactions
- Each user can have multiple transactions
- Each transaction belongs to exactly one user

**Indexing:**
- Primary index on _id for both collections
- Unique index on email in Users collection
- Compound index on (userId, date) in Transactions for efficient querying

## CHAPTER 6: IMPLEMENTATION

### 6.1 User Authentication Module

**Registration Process:**
1. User submits registration form with name, email, password
2. Frontend validates input fields
3. Backend receives registration request
4. Email uniqueness is checked in database
5. Password is hashed using bcrypt (10 salt rounds)
6. New user document is created in Users collection
7. Success response sent to frontend
8. User redirected to login page

**Login Process:**
1. User submits email and password
2. Backend searches for user by email
3. If user exists, password hash is compared
4. If match successful, JWT token is generated
5. Token includes user ID and expiration time
6. Token sent to frontend
7. Frontend stores token in localStorage
8. User redirected to dashboard

**Authentication Middleware:**
- Intercepts requests to protected routes
- Extracts JWT token from Authorization header
- Verifies token signature and expiration
- Decodes user ID from token
- Attaches user ID to request object
- Allows request to proceed or returns 401 error

### 6.2 Transaction Management Module

**Add Transaction:**
1. User fills transaction form on dashboard
2. Form data validated on client-side
3. POST request sent to /api/transactions endpoint
4. Backend validates required fields
5. Authentication token verified
6. New transaction document created with user ID
7. Document saved to Transactions collection
8. Response sent with created transaction data
9. Frontend updates dashboard displays

**View Transactions:**
1. Dashboard loads and requests transaction data
2. GET request to /api/transactions with auth token
3. Backend queries Transactions collection by user ID
4. Transactions sorted by date (newest first)
5. Data returned as JSON array
6. Frontend displays in transaction list
7. Each transaction shows type, amount, category, date

**Delete Transaction:**
1. User clicks delete button on transaction
2. Confirmation dialog shown
3. DELETE request sent with transaction ID
4. Backend verifies transaction belongs to user
5. Transaction removed from database
6. Success response sent
7. Frontend removes transaction from display
8. Dashboard totals automatically update

### 6.3 Dashboard and Analytics Module

**Dashboard Data Calculation:**
1. Backend receives request for dashboard stats
2. All user transactions fetched from database
3. Transactions filtered by type (income/expense)
4. Amounts summed using reduce function
5. Balance calculated (income - expenses)
6. Category-wise totals computed
7. Response includes:
   - totalIncome
   - totalExpense
   - balance
   - categoryTotals object
   - transactionCount

**Real-time Updates:**
- After each transaction operation (add/delete)
- Dashboard automatically refreshes statistics
- Chart redraws with updated data
- No page reload required
- Smooth user experience

### 6.4 Data Visualization Module

**Pie Chart Implementation:**
1. Category totals received from backend
2. Canvas element selected from DOM
3. 2D drawing context obtained
4. Total expense calculated for percentages
5. Color array defined for categories
6. For each category:
   - Calculate percentage and angle
   - Draw pie slice with arc
   - Add label with category name and percentage
7. Chart displays spending distribution visually

**Chart Features:**
- Dynamic colors for different categories
- Percentage labels for each slice
- Responsive sizing
- Clear legend showing category names
- Updates automatically when data changes

### Installation Guide

**Prerequisites:**
- Node.js (v14 or higher)
- MongoDB (v4.4 or higher)

**Step 1: Download Project**
- Download the project files
- Extract to a folder

**Step 2: Install Dependencies**
```
cd expense-tracker
npm install
```

**Step 3: Start MongoDB**
```
mongod
```

**Step 4: Start Server**
```
node server.js
```

**Step 5: Access Application**
- Open browser
- Navigate to http://localhost:3000

### User Guide

**Creating an Account:**
1. Click "Sign up" link on login page
2. Enter your name, email, and password
3. Click "Sign Up" button
4. You'll be redirected to login page

**Logging In:**
1. Enter your email and password
2. Click "Login" button
3. You'll be taken to the dashboard

**Adding Transactions:**
1. Fill out the transaction form on dashboard
2. Select type (Income or Expense)
3. Enter amount
4. Choose category
5. Select date
6. Add description (optional)
7. Click "Add Transaction"

**Viewing Transactions:**
- All transactions appear in the list below the form
- Each shows type, amount, category, description, and date

**Deleting Transactions:**
1. Find the transaction you want to delete
2. Click the "Delete" button
3. Confirm deletion
4. Transaction will be removed

**Understanding Dashboard:**
- Total Income: Sum of all income transactions (green)
- Total Expense: Sum of all expense transactions (red)
- Balance: Income minus Expenses (blue)
- Pie Chart: Visual breakdown of expenses by category

**Logging Out:**
- Click "Logout" button in navigation bar
- You'll be redirected to login page
